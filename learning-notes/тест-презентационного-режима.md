# Git: Основы системы контроля версий

*Презентация для начинающих разработчиков*

---

## 📋 Что такое Git?

### 🔧 **Git** — система контроля версий
- Отслеживает изменения в файлах
- Сохраняет историю проекта
- Обеспечивает командную работу

---

## ⚡ Основные команды

### 🚀 Базовый рабочий процесс

```bash
git init     # Создание репозитория
git add      # Добавление файлов
git commit   # Сохранение изменений
git push     # Отправка на сервер
```

---

## 🌿 Работа с ветками

### Зачем нужны ветки?
- ✅ **Параллельная разработка** функций
- ✅ **Изоляция** экспериментов
- ✅ **Безопасность** основной ветки

### Объединение веток
```bash
git merge    # Объединяет ветки
```
⚠️ **Конфликты** решаются вручную

## 💡 Преимущества Git

### 📚 **Сохранение истории**
- Каждое изменение записывается
- Можно посмотреть, кто что изменил
- Полная история проекта

### 👥 **Командная работа**
- Несколько человек работают одновременно
- Автоматическое слияние изменений
- Разрешение конфликтов

### ⏪ **Откат изменений**
- Возврат к любой предыдущей версии
- Отмена неудачных экспериментов
- Восстановление удаленных файлов
