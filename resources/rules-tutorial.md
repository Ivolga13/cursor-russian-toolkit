# Rules Tutorial - Your Automatic Guides

## What Are Rules?

Rules are like having a helpful teacher who automatically guides you toward good practices as you work. They check your files in real-time and provide gentle suggestions to keep your work consistent and professional.

## How Rules Work

Think of Rules as automatic quality checks:
- **Always active** - They work in the background while you type
- **File-specific** - Different Rules apply to different types of files
- **Gentle guidance** - They suggest improvements without being pushy
- **Consistent standards** - They help maintain the same quality across all your work

## Rules in Your Project

### 1. Markdown Formatting Rule

**What it does:** Ensures your markdown files follow consistent formatting standards

**Applies to:** All `.md` files (your notes and documentation)

**What it checks:**
- Proper heading hierarchy (# ## ###)
- Consistent bullet point formatting
- Required sections in learning notes
- Clear link formatting
- Appropriate emphasis usage

**Example guidance:**
- Suggests using `# Main Title` instead of `## Main Title` for document titles
- Reminds you to include "What I Learned" sections in learning notes
- Helps format links consistently: `[Clear description](url)`

### 2. File Naming Rule

**What it does:** Guides you toward consistent, searchable file names

**Applies to:** All files in your project

**What it enforces:**
- Lowercase letters with hyphens
- Descriptive names that explain content
- Date format: YYYY-MM-DD when relevant
- Proper file extensions

**Example guidance:**
- Suggests `cursor-basics.md` instead of `Cursor Basics.md`
- Recommends `daily-progress-2026-04-07.md` for dated files
- Helps organize files in appropriate folders

## How Rules Help You Learn

### Immediate Feedback
As you type, Rules provide gentle suggestions. This helps you:
- Learn good practices naturally
- Develop consistent habits
- Avoid common formatting mistakes
- Focus on content while maintaining quality

### Progressive Learning
Rules teach you standards gradually:
- Start with basic formatting
- Build good habits through repetition
- Learn why certain practices matter
- Develop your own sense of quality

## Understanding Rule Feedback

### Types of Suggestions

**Formatting Suggestions**
```
"Consider using ## for section headings instead of ###"
"Add a blank line before this heading for better readability"
```

**Structure Suggestions**
```
"Learning notes work best with a 'What I Learned' section"
"Consider adding a 'Next Steps' section to track your progress"
```

**Style Suggestions**
```
"Keep sentences under 25 words for better readability"
"Use active voice: 'I learned' instead of 'It was learned'"
```

### When to Follow Rule Suggestions

**Always follow:**
- File naming conventions
- Basic markdown formatting
- Required sections for specific document types

**Consider following:**
- Style suggestions that improve readability
- Structure recommendations for better organization
- Consistency improvements across documents

**You can ignore:**
- Suggestions that don't fit your specific context
- Style preferences that conflict with your voice
- Rules that don't apply to your current task

## Creating Your Own Rules

### When to Create Custom Rules

- You have specific standards for your learning process
- You want consistency in areas not covered by existing Rules
- You're working on specialized content with unique requirements
- You want to enforce personal preferences automatically

### Simple Rule Ideas for Beginners

1. **Learning Journal Format** - Ensure daily entries have consistent structure
2. **Resource Link Standards** - Format all external links consistently
3. **Question Tracking** - Require certain sections in question documents
4. **Progress Measurement** - Standard format for tracking learning milestones

### Basic Rule Structure

Every Rule needs:
1. **Clear scope** - Which files should this Rule check?
2. **Specific standards** - What exactly should be consistent?
3. **Helpful examples** - Show good and bad examples
4. **Reasonable flexibility** - Don't be too rigid

## Best Practices

### Working with Rules Effectively

1. **Start simple** - Begin with basic formatting Rules
2. **Learn gradually** - Add more Rules as you understand their value
3. **Stay flexible** - Rules should help, not hinder your creativity
4. **Review regularly** - Update Rules as your needs change

### Balancing Consistency and Creativity

- **Use Rules for structure** - Headings, formatting, organization
- **Keep content flexible** - Don't let Rules stifle your voice
- **Focus on readability** - Rules should make content easier to understand
- **Maintain your style** - Rules enhance your voice, they don't replace it

## Troubleshooting

### "Rules are too strict"
- Review which Rules are active for your file type
- Consider modifying Rules to be less restrictive
- Remember you can ignore suggestions that don't fit

### "I don't understand why a Rule is suggesting something"
- Look at the Rule documentation to understand the reasoning
- Try following the suggestion to see if it improves your work
- Ask for clarification about specific Rule recommendations

### "Rules conflict with each other"
- This usually indicates Rules need refinement
- Prioritize Rules that improve readability and consistency
- Consider combining or modifying conflicting Rules

### "Rules don't seem to be working"
- Check that your file type matches the Rule's scope
- Verify Rules are properly formatted and saved
- Remember that Rules provide suggestions, not automatic changes

## Practice Exercises

### Exercise 1: Formatting Practice
1. Create a new markdown file with intentionally poor formatting
2. Notice which Rules provide suggestions
3. Apply the suggestions and see how readability improves
4. Learn the patterns Rules are teaching you

### Exercise 2: File Naming Practice
1. Create files with various naming styles
2. Observe Rule suggestions for naming improvements
3. Rename files following the suggestions
4. Notice how consistent naming improves file organization

### Exercise 3: Structure Practice
1. Write notes without following any structure
2. Let Rules guide you toward better organization
3. Compare structured vs. unstructured notes
4. Develop your own sense of good document structure
