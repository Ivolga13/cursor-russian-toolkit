# FAQ — Часто задаваемые вопросы

*Ответы на самые популярные вопросы о Cursor Russian Toolkit*

## 🚀 Установка и настройка

### ❓ Как понять, что система установлена правильно?
**Ответ:** Создайте файл `.md` и напишите текст с ошибками типографики:
```
тест "кавычек" и тире - проверка
```
Если система предлагает исправления, всё работает корректно.

### ❓ Система не предлагает исправления. Что делать?
**Ответ:** Проверьте:
1. Файлы Rules находятся в `.cursor/rules/` и имеют расширение `.mdc`
2. Файлы Skills находятся в `.cursor/skills/[название]/SKILL.md`
3. Перезапустили ли вы Cursor после установки
4. Работаете ли в файле с расширением `.md`

**Важно**: Rules дают подсказки, но не изменяют файлы автоматически. Skills работают только по явному запросу.

### ❓ Можно ли использовать систему в существующем проекте?
**Ответ:** Да! Просто скопируйте папки `.cursor/rules/` и `.cursor/skills/` в корень вашего проекта. Система не влияет на существующий код.

### ❓ Работает ли система с другими языками?
**Ответ:** Система специально создана для русского языка, но базовые Skills (Writing Assistant, Content Organizer) работают с любыми текстами.

### ❓ Почему Rules не переименовывают мои файлы автоматически?
**Ответ:** Rules НЕ изменяют существующие файлы автоматически — это было бы слишком агрессивно. Они только:
- Предлагают рекомендации при создании новых файлов
- Показывают подсказки по улучшению
- Направляют к лучшим практикам

Чтобы переименовать файлы, сделайте это вручную согласно рекомендациям Rules.

### ❓ Почему Skills не работают автоматически?
**Ответ:** Skills работают ТОЛЬКО по явному запросу. Это сделано специально:
- Вы контролируете, когда применять улучшения
- Можете выбирать конкретные Skills для конкретных задач
- Избегаете нежелательных изменений

**Правильное использование**: "Используй Russian Text Enhancer для этого текста"

## 🤖 Использование Skills

### ❓ Как правильно обращаться к Skills?
**Ответ:** Используйте точные названия:
- `"Используй Russian Text Enhancer"`
- `"Примени Auto Processor в полном режиме"`
- `"Применить Content Organizer"`

### ❓ Skill не понимает мою команду. Что делать?
**Ответ:** Попробуйте:
1. Использовать точное название Skill
2. Быть более конкретным: "Используй Russian Text Enhancer для улучшения этого текста"
3. Разбить задачу на части: сначала один Skill, потом другой

### ❓ Можно ли использовать несколько Skills одновременно?
**Ответ:** Да! Auto Processor автоматически применяет несколько Skills в правильной последовательности. Или используйте их по очереди.

### ❓ Какой режим Auto Processor выбрать?
**Ответ:** 
- **Quick Mode** — для личных заметок (30 сек)
- **Full Mode** — для документации (2-3 мин)
- **Presentation Mode** — для презентаций (3-4 мин)

## 📝 Работа с текстами

### ❓ Система изменяет мой стиль письма?
**Ответ:** Нет, система улучшает качество, сохраняя ваш голос. Она исправляет ошибки и улучшает структуру, но не меняет смысл и тон.

### ❓ Что делать, если не согласен с предложениями системы?
**Ответ:** Вы полностью контролируете процесс! Применяйте только те изменения, которые считаете нужными. Система предлагает, вы решаете.

### ❓ Работает ли система с техническими текстами?
**Ответ:** Да! Система специально настроена для IT-контента:
- Правильно выделяет технические термины
- Форматирует код в блоки
- Работает со смешанным русско-английским контентом

### ❓ Можно ли обрабатывать очень длинные документы?
**Ответ:** Для больших документов рекомендуется:
1. Обрабатывать по разделам
2. Использовать Quick Mode для первичной обработки
3. Применять специализированные Skills к конкретным частям

## 🔧 Настройка и кастомизация

### ❓ Как настроить систему под свои предпочтения?
**Ответ:** Создайте файл `.cursor/my-preferences.md` с вашими настройками:
```markdown
# Мои настройки
- Стиль: неформальный
- Кавычки: «елочки»
- Технические термины: с переводами
```

### ❓ Можно ли создать собственный Skill?
**Ответ:** Да! Изучите структуру существующих Skills в папке `.cursor/skills/` и создайте аналогичный файл со своими инструкциями.

### ❓ Как отключить определенные Rules?
**Ответ:** Переименуйте файл Rule, добавив `.disabled` в конец:
```
russian-typography.mdc → russian-typography.mdc.disabled
```

### ❓ Можно ли использовать систему для команды?
**Ответ:** Да! Создайте общий репозиторий с настройками и попросите всех участников команды использовать одни и те же Rules и Skills.

## ⚡ Производительность

### ❓ Система работает медленно. Как ускорить?
**Ответ:** 
1. Начните с Quick Mode
2. Обрабатывайте документы по частям
3. Проверьте скорость интернет-соединения
4. Используйте конкретные команды вместо общих

### ❓ Сколько времени занимает обработка?
**Ответ:** 
- Quick Mode: 30 секунд - 1 минута
- Full Mode: 2-3 минуты
- Presentation Mode: 3-4 минуты
- Время зависит от размера документа

### ❓ Можно ли работать офлайн?
**Ответ:** Rules работают офлайн, но Skills требуют подключения к интернету для работы AI.

## 🐛 Решение проблем

### ❓ Skills перестали работать после обновления Cursor
**Ответ:** 
1. Проверьте, что файлы Skills остались на месте
2. Перезапустите Cursor
3. Попробуйте переустановить систему

### ❓ Система выдает ошибки на английском языке
**Ответ:** Это нормально — Cursor использует английский для системных сообщений. Если ошибка касается вашего контента, опишите проблему подробнее.

### ❓ Rules работают, но Skills не отвечают
**Ответ:** Проверьте:
1. Правильность структуры папок Skills
2. Наличие файлов SKILL.md
3. Корректность frontmatter в начале файлов
4. Стабильность интернет-соединения

### ❓ Система предлагает неправильные исправления
**Ответ:** 
1. Сообщите об этом в Issues репозитория
2. Приложите пример текста и неправильного исправления
3. Мы улучшим систему на основе вашего отзыва
