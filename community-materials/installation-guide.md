# Инструкция по установке Cursor Russian Toolkit

*Пошаговое руководство для быстрой установки и настройки*

## 📋 Требования

### Минимальные требования
- **Cursor IDE** версии 0.8 или новее
- **Операционная система**: Windows, macOS, или Linux
- **Место на диске**: 5 МБ для всех файлов системы

### Рекомендуемые требования
- Базовые знания работы с файлами и папками
- Понимание структуры проектов в Cursor
- Желание улучшить качество русскоязычных документов 😊

## 🚀 Быстрая установка (2 минуты)

### Шаг 1: Подготовка
1. Откройте ваш проект в **Cursor**
2. Убедитесь, что в корне проекта есть папка `.cursor`
3. Если папки нет — создайте её

### Шаг 2: Загрузка файлов
Скачайте следующие файлы из репозитория:

**Rules (правила):**
```
.cursor/rules/russian-typography.mdc
.cursor/rules/russian-writing-style.mdc  
.cursor/rules/mixed-language-content.mdc
.cursor/rules/workflow-automation.mdc
```

**Skills (навыки):**
```
.cursor/skills/russian-text-enhancer/SKILL.md
.cursor/skills/auto-processor/SKILL.md
.cursor/skills/writing-assistant/SKILL.md
.cursor/skills/content-organizer/SKILL.md
```

### Шаг 3: Размещение файлов
1. Скопируйте папку `rules` в `.cursor/rules/`
2. Скопируйте папку `skills` в `.cursor/skills/`
3. Структура должна выглядеть так:

```
ваш-проект/
├── .cursor/
│   ├── rules/
│   │   ├── russian-typography.mdc
│   │   ├── russian-writing-style.mdc
│   │   ├── mixed-language-content.mdc
│   │   └── workflow-automation.mdc
│   └── skills/
│       ├── russian-text-enhancer/
│       │   └── SKILL.md
│       ├── auto-processor/
│       │   └── SKILL.md
│       ├── writing-assistant/
│       │   └── SKILL.md
│       └── content-organizer/
│           └── SKILL.md
└── [остальные файлы проекта]
```

### Шаг 4: Активация
1. **Перезапустите Cursor** — это важно для загрузки новых Rules и Skills
2. Создайте тестовый файл `test.md`
3. Начните печатать — Rules должны автоматически начать работать

## ✅ Проверка установки

### Тест 1: Rules работают
Создайте файл и напишите:
```
тест "кавычек" и тире - проверка
```

Система должна предложить исправить на:
```
тест «кавычек» и тире — проверка
```

### Тест 2: Skills доступны
Напишите любой русский текст и попросите:
```
"Используй Russian Text Enhancer для улучшения этого текста"
```

Если система отвечает и улучшает текст — всё работает! 🎉

### Тест 3: Автоматизация активна
Попробуйте команду:
```
"Примени Auto Processor в быстром режиме"
```

Система должна автоматически улучшить весь документ.

## 🔧 Решение проблем

### Проблема: Rules не работают
**Симптомы:** Нет предложений по исправлению типографики

**Решение:**
1. Проверьте, что файлы `.mdc` находятся в `.cursor/rules/`
2. Убедитесь, что файлы имеют правильные расширения
3. Перезапустите Cursor
4. Попробуйте создать новый `.md` файл

### Проблема: Skills не отвечают
**Симптомы:** AI не понимает команды типа "Используй Russian Text Enhancer"

**Решение:**
1. Проверьте структуру папок Skills
2. Убедитесь, что файлы называются `SKILL.md`
3. Проверьте, что в начале каждого SKILL.md есть правильный frontmatter:
   ```yaml
   ---
   name: skill-name
   description: описание
   ---
   ```

### Проблема: Система работает медленно
**Симптомы:** Долгая обработка текстов

**Решение:**
1. Начните с простых команд: "Russian Text Enhancer"
2. Для больших документов используйте Skills по частям
3. Убедитесь, что у вас стабильное интернет-соединение

## 🎛️ Настройка под ваши нужды

### Персональные предпочтения
Создайте файл `.cursor/my-preferences.md`:

```markdown
# Мои настройки

## Стиль письма
- Тон: [формальный/неформальный/дружелюбный]
- Аудитория: [коллеги/студенты/широкая публика]
- Сложность: [простой/средний/экспертный]

## Типографика
- Кавычки: [«елочки» / „лапки"]
- Тире: [длинное — / среднее –]
- Числа: [10 000 / 10000]

## Автоматизация
- Режим по умолчанию: [Quick/Full/Presentation]
- Добавлять эмодзи: [да/нет]
- Технические термины: [с переводом/без перевода]
```

### Корпоративные стандарты
Для команд можно создать общий файл настроек:

```markdown
# Корпоративные стандарты

## Документооборот
- Все документы проходят через Auto Processor
- Обязательные разделы: Введение, Основная часть, Выводы
- Технические термины выделяются жирным шрифтом

## Презентации
- Использовать Presentation Mode
- Максимум 7 пунктов на слайде
- Обязательные эмодзи для визуальных акцентов
```
