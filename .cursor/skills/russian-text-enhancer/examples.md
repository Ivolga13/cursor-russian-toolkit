## Пример 3: Техническая инструкция

### Исходный текст
```
как установить docker
1. идем на сайт docker.com
2. скачиваем docker desktop
3. устанавливаем
4. проверяем что все работает командой docker --version
```

### После применения Russian Text Enhancer
```markdown
# Установка Docker

## Требования к системе
Перед установкой убедитесь, что ваша система поддерживает **Docker Desktop**.

## Пошаговая инструкция

### Шаг 1: Загрузка
1. Перейдите на официальный сайт [Docker](https://docker.com)
2. Найдите раздел «Download Docker Desktop»
3. Выберите версию для вашей операционной системы

### Шаг 2: Установка
1. Запустите скачанный установщик
2. Следуйте инструкциям мастера установки
3. Перезагрузите компьютер при необходимости

### Шаг 3: Проверка установки
Убедитесь, что **Docker** установлен правильно:

```bash
docker --version
```

Команда должна вывести версию установленного **Docker**.

## Возможные проблемы
- **Ошибка прав доступа**: запустите терминал от имени администратора
- **Конфликт с антивирусом**: добавьте Docker в исключения
- **Проблемы с виртуализацией**: включите Hyper-V в настройках Windows

## Следующие шаги
После успешной установки можно переходить к изучению основных команд **Docker**.
```

## Пример 4: Смешанный русско-английский контент

### Исходный текст
```
работа с API
сегодня изучала REST API. делала GET и POST requests к серверу.
использовала библиотеку requests в python.
получила JSON response с данными пользователей.
```

### После применения Russian Text Enhancer
```markdown
# Работа с API

## Изучаемые технологии
Сегодня изучала **REST API** — архитектурный стиль для веб-сервисов.

## Практические задачи
Выполняла различные типы запросов к серверу:
- **GET requests** — получение данных
- **POST requests** — отправка данных

## Используемые инструменты
Работала с библиотекой `requests` в **Python**:

```python
import requests

# GET запрос
response = requests.get('https://api.example.com/users')
data = response.json()
```

## Результаты
Успешно получила **JSON response** с данными пользователей и научилась обрабатывать структурированные данные.

## Планы на изучение
- Изучить аутентификацию в **API**
- Освоить обработку ошибок
- Попробовать работу с **GraphQL**
```

## Типичные улучшения

### Типографические исправления
- `"кавычки"` → `«кавычки»`
- `слово - слово` → `слово — слово`
- `2020-2025` → `2020–2025`
- `10000` → `10 000`
- `т.д.` → `т. д.`

### Стилистические улучшения
- Канцеляризмы → живой язык
- Длинные предложения → короткие и ясные
- Пассивный залог → активный залог
- Общие фразы → конкретные формулировки

### Структурные улучшения
- Добавление заголовков и подзаголовков
- Создание логичной структуры
- Форматирование списков
- Выделение ключевых терминов
- Добавление примеров кода в блоки
